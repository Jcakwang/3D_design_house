# PRD 需求文档：3D 看房工具

> **文档版本**: v1.0  
> **日期**: 2026-06-08  
> **角色**: 产品经理（AI）  
> **项目名称**: 3D 看房工具（3d-house-viewer）

---

## 1. 项目背景

现在很多人买房，尤其是新房，基本是看线上信息。但现在的看房网站有个问题：户型图是平面的，看了半天还是不知道房子实际住进去什么感觉。

我们想做的是一个 **网页端的 3D/VR 看房工具** — 用户不用装任何 App，打开网页就能像玩游戏一样"走进"一个房子里看房。就像《模拟人生》那种感觉，但看的是真实的房子。

**为什么现在做？**  
因为 3D 技术越来越成熟，成本也越来越低，普通用户完全可以用手机或电脑体验。这对房产平台来说是一个很好的差异化功能，也能吸引更多用户。

---

## 2. 目标用户

| 用户类型 | 说明 |
|---------|------|
| **主要用户**：正在看房买房的普通人 | 他们可能第一次买房，对户型没概念，想提前看看房子实际住进去什么样子 |
| **次要用户**：房产中介/开发商 | 他们可以用这个工具给客户提供更好的展示，增加成交概率 |
| **不太需要**：已经买房的人 | 这个工具对购房阶段的人最有价值 |

**用户画像举例**：  
张伟，32 岁，第一次买房，在网上看了很多户型图，但完全想象不出来这个房子实际住进去什么感觉，每天都很纠结。

---

## 3. 用户痛点

1. **户型图看不懂**  
   平面图太小太抽象，旋转、采光、空间感完全靠想象。

2. **线上看房不真实**  
   现在房产网站上的"VR 看房"大多是 360 度全景照片拼起来的，很僵硬，不能自由走动。

3. **不知道装修后的样子**  
   很多房子是毛坯房（还没装修的），用户不知道怎么装修，想象不出来。

4. **对比户型不方便**  
   看了好几个小区、好几个户型，记不住每个户型的特点，没法比较。

5. **看房效率低**  
   有些房子在很远，先花半天时间跑过去，发现根本不是自己想要的，浪费时间。

---

## 4. 核心使用流程

```
打开网页
  │
  ▼
📍 地图上看到各个小区（带标记点）
  │
  ▼
🏘️ 点击某个小区 → 看到该小区所有楼盘和户型列表
  │
  ▼
🏠 点击某个户型 → 进入 3D/VR 看房页面
  │
  ▼
🎨 切换装修风格（毛坯 / 现代简约 / 北欧 / 中式...）
  │
  ▼
❤️ 收藏喜欢的户型 → 回列表对比
```

**一句话总结**：从地图找小区 → 选户型 → 进 3D 看房 → 换装修 → 收藏

## 5. 第一版 MVP（最小可行版本）功能

MVP 的意思是：先做一个能用、能演示的最小版本，不需要什么功能都有，但核心体验要完整。

### 5.1 地图页面
- 展示一个简化地图（可以用高德地图的网页版嵌入）
- 地图上标记几个小区的位置（先用 5-10 个示意数据）
- 点击标记点，弹出小区名称

### 5.2 小区详情页
- 显示该小区的基本信息（名称、均价、户型数量）
- 列出该小区的几个户型卡片（卡片上有户型名称、面积、几张示意图）

### 5.3 户型详情页（3D/VR 看房页面）
- 展示户型的 3D 模型，用户可以拖拽视角、平移、放大缩小
- 可以切换不同的装修风格（点击按钮切换）
- 有一个"收藏"按钮，点一下就把户型加入收藏

### 5.4 收藏夹
- 一个列表页面，展示用户收藏的户型
- 可以从这里再次进入 3D 看房

## 7. 数据从哪里来

第一版 MVP 不需要真实数据，可以用 **示意数据（模拟数据）**。

| 数据类型 | 来源 | 说明 |
|---------|------|------|
| 小区信息 | AI / 手动生成 | 小区名、位置（经纬度）、均价、户型数量 — 全部是假的，用于演示 |
| 户型信息 | AI / 手动生成 | 户型名、面积、房间数、描述 — 假的也没关系 |
| 3D 模型 | AI 生成 / 简化模型 | 这是核心，后面展开说 |
| 装修风格 | AI 生成 | 模型换颜色、换贴图（材质的图片） |
| 地图底图 | 高德地图 API | 这个需要接入真实的地图服务（免费额度够用了） |

**第一版建议**：  
全部用示意数据。地图用真实的（高德地图），但上面的小区标记和户型全部自己编。这样演示时效果已经很好了。

## 8. 哪些内容可以用 AI 生成示意数据

这部分是 3D 看房工具里最耗时的部分 — 真实的 3D 模型不是随便就能做出来的。所以第一版我们用 AI 来生成：

| 数据项 | AI 生成方式 | 说明 |
|-------|------------|------|
| **小区名称和基本信息** | AI 写一段文字 | 让 AI 生成 5-10 个虚构小区的名字、面积、价格 |
| **户型信息** | AI 生成 JSON 数据 | 让 AI 按格式生成户型数据，比如 `{"name":"三室两厅", "area":120, "rooms":3}` |
| **户型缩略图** | AI 画图 | 让 AI 生成户型的平面图截图（类似户型图） |
| **3D 房间模型** | **用 AI 生成的简单模型** | 不用真正的 3D 建模，第一版可以用几个基础盒子（代表墙、地板）拼起来就行 |
| **装修风格** | AI 生成颜色/材质参数 | 每个装修风格对应一套颜色值，比如"现代"是灰色+白色，"北欧"是米色+浅木色 |

### 关于 3D 模型的特别说明

**真实情况**：一个真正的 3D 模型需要专业建模师花好几天来做。  
**MVP 的做法**：我们用一个简单的方案 — 用代码自动生成房间的基本结构（地板、四面墙），然后贴上不同颜色的贴图来模拟装修风格。这样不需要专业建模师，前端代码就能搞定。

**比喻**：就像乐高积木，用几个基础形状拼出房间的样子，然后给它们"换衣服"（换颜色）来表示不同装修。

## 9. 哪些功能以后再做（第二版及以后）

### 第二版可以做的
| 功能 | 说明 |
|------|------|
| **用户登录** | 用手机号或微信登录，收藏数据存到云端，换设备也能看到 |
| **真实小区数据接入** | 对接真实的房产数据 API（比如链家、安居客的接口） |
| **更精细的 3D 模型** | 加入真实的户型图纸，生成更准确的 3D 模型 |
| **多人 VR** | 两个人可以同时在一个虚拟房子里看房，还能聊天 |
| **户型对比** | 选两个户型放一起比较 |

### 第三版可以做的
| 功能 | 说明 |
|------|------|
| **AI 智能推荐户型** | 根据用户的偏好（喜欢几室、预算多少）自动推荐户型 |
| **AI 装修建议** | 用户说"我喜欢日式风格"，AI 自动生成一个日式的装修方案 |
| **AR 看房** | 用手机摄像头对着地面，显示虚拟房子（增强现实） |
| **VR 头显支持** | 连接 Oculus 等 VR 眼镜，沉浸式看房 |
| **开发商后台** | 让开发商自己上传楼盘数据，管理自己的 3D 户型 |

### 暂时不考虑的
| 功能 | 原因 |
|------|------|
| 小区内部的 3D 漫游（走廊、电梯、大堂） | 工作量太大，第一版聚焦在单个户型 |
| 二手房真实房源 | 数据来源复杂，先做新房（开发商数据统一） |
| 估价系统 | 需要大量真实交易数据，先不做 |
| 贷款计算器 | 跟 3D 看房关系不大，可以加在第二版或单独做 |

## 10. 可能的风险和注意事项

### 技术风险
| 风险 | 说明 | 应对方案 |
|------|------|---------|
| **3D 模型加载慢** | 如果模型太大，用户在手机上可能打不开或者卡 | 第一版用简单的模型（几个盒子），不要一开始就做复杂的 |
| **手机端体验差** | 3D 看房在电脑上很流畅，但在手机上可能会卡顿 | 测试手机的性能，如果不行就限制功能（比如手机端不支持切换装修） |
| **浏览器兼容性** | 有些浏览器不支持 3D 功能 | 主流浏览器（Chrome、Safari、Edge）都没问题，提醒用户用这些 |

### 数据风险
| 风险 | 说明 | 应对方案 |
|------|------|---------|
| **户型数据不准确** | 如果用了真实数据，但数据过时了 | 第一版全部用示意数据，不担心这个问题 |
| **地图 API 限制** | 高德地图有调用次数限制 | 第一版数据量小，不会有这个问题 |

### 用户体验风险
| 风险 | 说明 | 应对方案 |
|------|------|---------|
| **用户不知道怎么用 3D 看房** | 有些用户没玩过 3D 的东西，可能不会操作 | 加一个简短的引导教程（3 个步骤的提示） |
| **装修切换效果不明显** | 如果换装修只是换颜色，用户可能觉得没变化 | 至少换 2-3 种明显的风格，让用户有感觉 |

### 法律风险
| 风险 | 说明 | 应对方案 |
|------|------|---------|
| **用了真实的楼盘数据** | 有些数据可能有版权 | 第一版用虚构数据，不做商业运营，没问题 |
| **截图被滥用** | 用户可能截图 3D 看房的内容去别的地方用 | 加个免责声明，说明内容是演示用的 |

## 开发顺序建议

这是按照 **先做完能演示、再做增强** 的顺序排下来的：

### 第一阶段：基础框架（1-2 周）
1. 搭一个网页项目的基本框架（前端框架用 React 或 Vue 都可以）
2. 做地图页面，接入高德地图 API
3. 在地图上放几个假的小区和标记点

### 第二阶段：核心流程（2-3 周）
4. 做小区详情页，展示户型卡片列表
5. 做 3D 看房页面，用代码生成简单的房间模型（地板+四面墙）
6. 做 3D 模型的旋转、缩放、平移功能
7. 做装修风格的切换功能

### 第三阶段：收藏功能（1 周）
8. 做收藏夹页面
9. 实现收藏/取消收藏功能
10. 收藏数据存在浏览器本地（不用数据库，第一版够用）

### 第四阶段：优化打磨（1 周）
11. 加一个简短的使用引导
12. 在手机上测试，修一下显示问题
13. 加一个"加载中"的动画，让体验更好
14. 整体测试一遍，准备演示

---

> **总结**：第一版的 MVP 核心就三件事 — 地图找小区、选户型进 3D 看房、切换装修。先把这三件事做好，其他都是锦上添花。
